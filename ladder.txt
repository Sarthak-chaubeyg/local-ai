QUESTION STYLE RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. BOARD-STYLE WORD PROBLEMS ONLY:
   - Questions should look like normal board-exam word problems.
   - Use natural language and realistic problem statements.
   - Do NOT use point-wise condition lists such as:
     • condition 1
     • condition 2
     • condition 3
   - Do NOT make the question feel like a logic puzzle or a bullet-point trap.
   - The wording should feel like a real school exam question, not a test of format.

2. PROGRESSIVE DIFFICULTY:
   - Start from absolute basics.
   - Then gradually increase difficulty.
   - The full set must move through:
     • basic understanding
     • standard application
     • deeper reasoning
     • hard application
     • JEE Mains-style reasoning
     • JEE Advanced-style reasoning

3. SYLLABUS BOUNDARY:
   - Use ONLY the attached chapter PDF as the source of truth.
   - Do NOT use any concept, formula, theorem, or idea outside the PDF.
   - If a concept is not present in the PDF, do not use it.

4. CONCEPT COVERAGE:
   - Cover all major topics and subtopics of the chapter.
   - Questions must collectively span the full chapter.
   - Do not leave important areas untouched.

5. MULTI-LEVEL THINKING:
   - Basic questions should test direct understanding.
   - Moderate questions should test application.
   - Hard questions should test analysis.
   - Mains questions should test multi-step reasoning.
   - Advanced questions should test deep, non-obvious reasoning.
   - The set should feel like a full mastery ladder.

6. NO SIMILARITY:
   - Every question must be structurally different.
   - Avoid repetitive patterns.
   - Avoid asking the same kind of question again and again with only numbers changed.

7. NO OUT-OF-SYLLABUS COMPLEXITY:
   - Keep everything within the chapter.
   - Do not import higher-class ideas.
   - Do not overcomplicate using concepts not in the PDF.

   - Check every step internally.
   - Re-solve every problem independently before showing it.

10. SELF-RE-EVALUATION:
    - For each question, do an internal audit before outputting:
      • syllabus correctness
      • conceptual correctness
      • difficulty level correctness
      • answer consistency
      • ambiguity check
      • wording clarity

    - If both paths do not match exactly, do not output the question.
    - Only output questions whose answers are fully consistent and verified.

12. NO GUESSING:
    - If a question cannot be solved with complete certainty, do not include it.
    - Prefer accuracy over quantity.
